# 游戏循环设计

## 文件定位

本文件定义项目所有核心循环。

包括：

* 秒级循环
* 分钟级循环
* 小时级循环
* 日循环
* 周循环
* 月循环

所有系统设计必须服务于循环体系。

---

# 设计目标

回答：

玩家为什么继续点击？

玩家为什么继续成长？

玩家为什么明天还回来？

玩家为什么一个月后还在玩？

禁止开发。

---

# 最终原则

任何功能必须进入循环。

不能进入循环的功能：

禁止开发。

循环大于功能。

成长大于内容。

留存大于复杂度。
